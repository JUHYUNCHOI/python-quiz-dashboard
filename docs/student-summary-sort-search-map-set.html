C++ STL 정리 — sort · 탐색 · map · set

한 장으로 정리. "이 도구 어디 쓰는 거였지?" 헷갈릴 때 펼쳐 봐요.

1. sort — 줄 세우기

sort(v.begin(), v.end()) 한 줄이면 끝. 오름차순.

// 오름차순 — 기본
vector<int> v = {5, 2, 9, 1, 7};
sort(v.begin(), v.end());
// v = {1, 2, 5, 7, 9}

// 내림차순 — greater 한 번 더
sort(v.begin(), v.end(), greater<int>());
// v = {9, 7, 5, 2, 1}
💡 정렬은 끝이 아니라 시작. 정렬해두면 그 다음에 할 수 있는 일들이 폭발적으로 늘어요 — 다음 섹션부터 그게 뭔지 봐요.

2. 정렬된 데이터에서 빠른 탐색 — 이진탐색 3 형제

정렬된 vector 에서만 동작. 안 정렬되어 있으면 엉뚱한 답 이 나와요.

함수답하는 질문돌려주는 것
binary_search(v.begin(), v.end(), x)"x 가 있나?"true / false
lower_bound(v.begin(), v.end(), x)"x 이상 의 첫 위치?"위치 (iterator)
upper_bound(v.begin(), v.end(), x)"x 초과 의 첫 위치?"위치 (iterator)

그림으로 — 값 3 찾기

{1,  3,  3,  5,  7,  9}
 0   1   2   3   4   5
     ↑       ↑
lower_bound  upper_bound
  (3 시작)   (3 끝 다음)

이터레이터는 그냥 "위치 손가락"

lower_bound 가 돌려주는 건 숫자가 아니라 위치 예요. 두 가지 공식만 외우면 끝:

auto it = lower_bound(v.begin(), v.end(), 3);

cout << *it;             // 그 위치의 값
cout << it - v.begin();   // 인덱스 숫자

활용 패턴 3 가지

vector<int> v = {1, 3, 3, 5, 7, 9};

// ① 값이 몇 개?  (upper - lower)
int cnt = upper_bound(v.begin(), v.end(), 3)
        - lower_bound(v.begin(), v.end(), 3);
// 3 가 몇 개? → 2 개

// ② 있는지만 확인 → binary_search 가 가장 깔끔
if (binary_search(v.begin(), v.end(), 3)) cout << "있음";

// ③ 위치가 필요할 때 → lower_bound
int idx = lower_bound(v.begin(), v.end(), 3) - v.begin();
⚠️ "있는지만" 확인이면 binary_search 만 써.
lower == upper 같은 트릭은 헷갈리니까 쓰지 말 것. 의도가 안 보임.
✅ "x 이상 첫 거" 같은 경계 질문은 lower_bound 가 유일한 도구.
예: 정렬된 점수 배열에서 70 점 이상 첫 학생 → lower_bound(v.begin(), v.end(), 70).

3. 중복 제거 — sort + unique + erase

중복 제거는 3 단계가 아니라 한 줄 이디엄 으로 외워요.

vector<int> v = {3, 1, 3, 2, 1, 3};

sort(v.begin(), v.end());                          // {1,1,2,3,3,3}
v.erase(unique(v.begin(), v.end()), v.end());     // {1,2,3}

unique 는 인접한 중복 만 제거해요. 그래서 sort 가 반드시 먼저.

💡 unique 가 직접 vector 크기를 못 줄여요 (이건 STL 규칙 — 알고리즘은 크기 못 바꿈). 그래서 erase 가 짝꿍. 이 한 줄을 통째로 외워두면 항상 같이 따라옴.

4. map — "이름 → 값" 빠른 검색표

Python dict 와 거의 같아요. 이름 (key) 으로 값을 빠르게 찾고 싶을 때.

// 만들기 + 값 넣기
map<string, int> scores;
scores["Alice"] = 95;
scores["Bob"] = 87;

이 4 가지만 알면 거의 다 풀림

// ① 있는지 확인
if (scores.count("Alice") > 0) {
    cout << "있어요!";
}

// ② 값 읽기 — 있을 때만 (없으면 0 이 자동 생성됨!)
if (scores.count("Bob") > 0) {
    cout << scores["Bob"];   // 87
}

// ③ 삭제
scores.erase("Bob");

// ④ 크기 / 비었나
cout << scores.size();    // 1
cout << scores.empty();   // 0 (false, 안 비었음)
⚠️ m["없는키"] 바로 접근하면 0 이 자동 생성 돼요.
반드시 count > 0 로 먼저 확인 → 있을 때만 m[key].

map 은 왜 sort 안 해도 빨라?

vector 는 sort 직접 호출해야 빠른 검색됐죠. map 은 다름:

📌 map 은 값을 넣을 때마다 알아서 정리해요. 그래서 따로 sort 안 해도 항상 빠른 상태.
→ 그냥 m.count(key) 쓰면 빠름. 안심하고 쓰면 됨.

map 순회 — pair 로 (key, value) 같이

for (auto& [name, score] : scores) {
    cout << name << ": " << score << endl;
}
// map 은 자동 정렬되어 있어서 알파벳 순으로 나옴:
// Alice: 95
// Bob: 87

순회 중 삭제 — 위험! 안전한 패턴

// ❌ 위험 — 돌면서 동시에 지우면 코드 꼬임
for (auto& [k, v] : m) {
    if (v < 0) m.erase(k);    // 위험!
}

// ✅ 안전 — 2 단계로 나누기
vector<string> toDelete;
for (auto& [k, v] : m) {
    if (v < 0) toDelete.push_back(k);
}
for (auto& k : toDelete) {
    m.erase(k);
}
💡 모은 다음 지우기. 이게 핵심.

5. set — 중복 없는 모음

map 에서 value 가 없어진 것 이 set. "있나/없나" 만 따지는 모음집.

set<int> nums;

nums.insert(5);          // 넣기 (중복은 자동 무시)
nums.insert(3);
nums.insert(5);          // 이미 있음 → 그대로

if (nums.count(5)) {     // 있나?
    cout << "있어요!";
}

nums.erase(3);           // 지우기
cout << nums.size();     // 원소 수

set 은 자동 정렬 되어 있어서 순회하면 작은 값부터 나와요.

for (int x : nums) cout << x << " ";
// 5 만 남아있음 → 5

set 의 진짜 쓸모 — 중복 제거 + "본 적 있나?" 추적

// 본 단어 추적
set<string> seen;

for (auto& w : words) {
    if (seen.count(w)) {
        cout << "중복!";
    } else {
        seen.insert(w);
    }
}

6. 헷갈리는 두 가지 count

이게 진짜 헷갈려서 따로 정리. 같은 이름이지만 완전히 다른 함수.

std::count(v.begin, v.end, x) m.count(key)
어디 거? algorithm 함수 (외부 도구) map / set 의 멤버 함수
어디서 씀? vector 같은 일반 범위 map, set
속도 O(N) — 처음부터 끝까지 훑음 O(log N) — 내부 트리 따라 점프
답하는 것 정확히 x 인 거 몇 개 키가 있나 (있으면 1, 없으면 0)
💡 한 줄로: std::count 는 vector 용 (느림). m.count 는 map/set 용 (빠름). 같은 이름인데 다른 함수.

📋 마지막 — 한 줄 치트시트

하고 싶은 일도구
줄 세우기 (오름차순)sort(v.begin(), v.end())
줄 세우기 (내림차순)sort(v.begin(), v.end(), greater<int>())
정렬된 vector — 있나?binary_search(v.begin(), v.end(), x)
정렬된 vector — x 이상 첫 위치lower_bound(v.begin(), v.end(), x)
정렬된 vector — x 가 몇 개?upper_bound - lower_bound
중복 제거sort 후 v.erase(unique(...), v.end())
map / set 에서 키 있나?m.count(key) > 0
map 에서 값 안전하게 읽기count 확인 → m[key]
map / set 삭제m.erase(key)
map / set 크기m.size() / m.empty()
🎯 큰 그림 한 줄로:
vector = 직접 sort 한 다음 algorithm 함수 (binary_search, lower_bound) 로 빠른 탐색.
map / set = 알아서 정리되는 컨테이너. 그냥 .count, .erase 만 호출하면 빠름.
