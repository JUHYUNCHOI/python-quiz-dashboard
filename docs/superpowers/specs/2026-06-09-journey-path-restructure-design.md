# 학습 여정 재구성 — "길(Guided Path)" 모델 설계

> 작성: 2026-06-09 · 상태: 설계 합의 완료 (브레인스토밍) · 다음: 단계별 구현 계획
> 트리거: cpp-17~20 "참고용" 을 학생이 안 봄 → 더 큰 문제(결정 피로) 발견
> 기준 문서: `memory/learning_tracks.md` (비전), `memory/cpp_curriculum_restructure_2026-04-29.md`

## 문제 & 북극성

**문제:** 지금 `/curriculum` 은 "강의 카탈로그(골라 담기)" 모델 — 연습/알고리즘/대회가 *동급 목적지 탭* 이고, "메인 트랙 끝" 뒤에 버튼 3개 + 참고용 레슨마다 [시작] 버튼이 쌓여 *선택을 강요*한다. 학생: "뭘 눌러야 하지?"

**북극성 (사용자 직접):** *"내가 뭘 해야할지 고민하는 게 아니라, 그냥 쭉쭉 가면서 풀어가면서 대회 문제까지 자연스럽게."* = **결정 피로 0.** 비전의 심장("다음 1개가 항상 큰 버튼")과 동일.

## 핵심 모델: 카탈로그 → "길(자동재생)"

학생은 메뉴를 안 본다. 앱은 항상 **딱 다음 1개**를 알고 그걸 보여준다. 끝 → 다음 → 끝 → 다음 … → 대회.

## 합의된 결정 (브레인스토밍)

1. **한 줄 척추(spine):** 트랙마다 *모든 것*(레슨·연습·코딩뱅크·알고리즘 토픽·대회 문제)을 하나의 순서 리스트로. "연습/알고리즘/대회" 단계 이름은 **구분선**일 뿐 갈림길 아님.
2. **커서 1개:** 앱은 "지금 어디"만 기억. `다음()` = 다음 미완료 항목.
3. **선수과목은 길 위 제자리 항목:** `cpp-18`(stack/queue) = 알고리즘 BFS/DFS **직전**, `cpp-19`(file I/O) = 대회 진입 **직전**. → "참고용 안 봄" 문제 자동 소멸.
4. **진짜 선택 콘텐츠는 길 밖:** `cpp-17`(STL탐색, Bronze~0%)·`cpp-20`(CP팁)·추가연습 = 조용한 "자료실 / 더 풀기" 로만. 절대 안 떠밂.
5. **화면 3개뿐:**
   - **홈 = 이어서 하기** — "⭐ 지금: [항목]" 카드 1개 + 큰 버튼 하나. 그게 끝.
   - **항목 화면** — 기존 레슨/문제 페이지 그대로.
   - **끝 = 다음 →** — 완료 후 [다음 →] 하나로 자동 다음 항목.
6. **탭 → "전체 지도" 로 강등:** 연습/알고/대회 탭(고르는 메뉴)은 정문에서 내려가고, "전체 지도 / 자료실" 로 *접어서* 항상 1탭 거리에 둠. 콘텐츠·페이지 **잃는 것 0**. 도구 네비(틀린문제·프로필)는 유지.
7. **길=기본, 자유=항상 가능:**
   - **뒤로(복습):** 완료한 것 지도에서 아무 때나 재방문 — 항상 자유.
   - **앞으로 점프(미리보기):** 지도에서 가능. 아직 도달 안 한 곳은 **부드러운 안내**(선택지 "나"): *"아직 stack/queue 안 봤어요 — 먼저 볼래요? [먼저 보기]/[그냥 갈래요]"*. 막지 않음.
8. **학생 선택 = 맨 처음 트랙(A/B/C) 하나뿐.** 그 뒤론 0.

## 절대 제약 (학생 데이터 보호)
- `lesson_id`, `completedLessons`/`completedQuizzes`/`question-mastery` localStorage 키, Supabase `lesson_progress` **무손상**. 길은 *기존 진도 위에 얹는 뷰/순서*일 뿐, 진도 저장 방식은 안 건드림.
- 어떤 콘텐츠/문제도 삭제하지 않음 (재배치/재노출만).

## 컴포넌트 (구현 단위)
- **척추 데이터** `lib/journey/spine-<track>.ts` — 순서 있는 항목 리스트 (각 항목: kind[lesson/practice/bank/algo/contest], id, 표시정보, 선수 위치).
- **다음-1개 엔진** `lib/journey/next-step.ts` — `getNextStep(progress, track)` = 첫 미완료 항목. 진도는 기존 소스에서 읽음.
- **`/journey` 홈** — 이어서 하기 카드(다음 1개) + 얇은 진도바 + "전체 지도" 링크.
- **끝 → 다음 hook** — 레슨/연습/문제 완료 화면에 [다음 →] (engine 사용).
- **전체 지도 뷰** — 기존 `/curriculum` 을 "지도" 로 (강등). 완료=자유, 잠긴 앞=부드러운 안내.
- **트랙 선택** — 최초 1회.

## 단계별 범위 (phasing)
- **Phase 1 (MVP, 보여줄 수 있는 슬라이스):** C++(Track C) 척추 1개 작성 + `/journey` 홈(이어서 하기 + 다음 1개, 기존 진도 읽음) + "전체 지도" 링크. → 학생이 "쭉쭉" 핵심을 체감.
- **Phase 2:** 각 항목 끝에 "다음 →" 붙이기 (레슨/연습/문제 완료 화면).
- **Phase 3:** Python(Track A·B) 척추 + 트랙 선택 UI.
- **Phase 4 (미래):** 적응형 — 틀리면 보충 자동 삽입 / 잘하면 건너뛰기.

## YAGNI (지금 안 함)
- 적응형 삽입(Phase 4) v1 에서 제외.
- 콘텐츠 삭제/대규모 데이터 마이그레이션 없음.
- 전체 맵 화려한 게임맵 비주얼은 후순위 (기능 먼저).

## 미해결 (구현 계획에서 확정)
- C++ 척추의 *정확한* 항목 순서 (특히 코딩뱅크 함수형 vs 입출력형 위치 — `contest_prep_flow` 메모 참고: 함수형 먼저, 입출력은 대회 직전).
- 앞-점프 부드러운 안내의 정확한 문구/동작.
- `/journey` 가 새 정문이 될 때 기존 `/curriculum` 링크/네비 정리 범위.
